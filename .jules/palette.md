## 2024-05-21 - Accessible Toggle Buttons for Password Visibility
**Learning:** When using non-native interactable elements (like `<i>` with FontAwesome icons) as toggle buttons (e.g., password visibility), applying `role="button"` and `tabindex="0"` is not enough. The `aria-label` must also be dynamically updated via JavaScript to accurately reflect the state (e.g., changing between "Tampilkan password" and "Sembunyikan password"), and explicit `keydown` listeners must be added to handle 'Enter' and 'Space' keys, ensuring true keyboard navigability and accurate screen reader announcements in the application's localized language.
**Action:** When creating or fixing custom UI controls, always verify that ARIA attributes are programmatically updated in sync with visual state changes, and that all non-native interactive elements fully support keyboard interactions.
