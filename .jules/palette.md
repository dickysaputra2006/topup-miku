## 2024-05-18 - Keyboard Accessibility for Custom Toggles
**Learning:** Playwright testing keyboard events (Enter/Space) on non-native interactive elements (like `<i>` used as buttons) in a headless environment often fails to register standard `.press()` actions correctly, requiring direct Javascript `dispatchEvent` simulation to verify correctly. Similarly, users relying on keyboard navigation face issues interacting with these elements unless explicitly programmed to listen for 'Enter' and 'Space' keydown events and trigger the click function.
**Action:** When converting non-native elements to buttons (`role="button"`, `tabindex="0"`), always attach a dedicated `keydown` listener that catches `e.key === 'Enter' || e.key === ' '` to manually invoke the toggle functionality. Additionally, ensure automated Playwright tests verify this keyboard capability using `page.evaluate('node => node.dispatchEvent(new KeyboardEvent("keydown", { key: "Enter" }))')` instead of relying solely on `page.locator.press("Enter")`.
