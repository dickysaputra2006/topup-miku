## 2026-05-10 - Adding missing accessibility to non-native interactive elements
**Learning:** Found a pattern in this application where non-native elements (such as `div` and `i`) are being used as interactive buttons without corresponding accessibility attributes. Specifically, `#notification-container` was relying exclusively on click listeners without a `tabindex`, `role="button"`, or keyboard event handlers. Screen readers and keyboard users would not be able to operate these components.
**Action:** When adding or updating interactive elements that don't natively receive focus (unlike `<button>` or `<a>`), always explicitly ensure they have `role="button"`, `tabindex="0"`, a descriptive `aria-label`, and `keydown` event handlers mapped to both the `Enter` and `Space` keys.
