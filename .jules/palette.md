## 2025-05-25 - [Accessibility Enhancements for Interactive Elements]
**Learning:** Found that many non-button interactive elements (e.g. `<i>` for password toggle, notification bells) lacked keyboard support, `role="button"`, and `tabindex="0"`, making them inaccessible to keyboard and screen reader users. Furthermore, complex UI components like the notification bell wrapped in a container required extracting the clickable area into a dedicated child trigger element to prevent creating an invalid ARIA structure. Also, it's critical to ensure all labels correctly match the application's localization (Indonesian).
**Action:** Always add `role="button"`, `tabindex="0"`, and event listeners for both `click` and `keydown` (Enter/Space) when making non-native elements interactable. For complex structures with dropdown panels, separate the trigger from the content container. Respect local language strings.
