## 2025-05-19 - Missing SSRF Validation in Outbound Webhooks
**Vulnerability:** A Server-Side Request Forgery (SSRF) vulnerability via DNS rebinding existed in `backend/utils/cronUtils.js`. Webhooks configured with user-controlled `h2h_callback_url` did not securely validate resolved IPs against private network ranges.
**Learning:** Checking the URL hostname string is insufficient for SSRF protection because attackers can configure a domain to point to a public IP during registration and rebind it to an internal IP (like `127.0.0.1` or `169.254.169.254`) right before the server executes the outbound webhook.
**Prevention:** Override the `dns.lookup` function via custom `http.Agent` and `https.Agent` when using HTTP clients like Axios. This ensures that the IP validation happens directly on the resolved IP at the exact moment of connection, effectively preventing TOCTOU DNS rebinding SSRF attacks. Also remember to explicitly block the `0.0.0.0/8` range to prevent loopback bypasses.
