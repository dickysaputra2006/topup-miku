## 2024-05-24 - SSRF via DNS Rebinding in Webhooks
**Vulnerability:** The application was vulnerable to Server-Side Request Forgery (SSRF) and DNS Rebinding when performing outbound requests (e.g., webhook callbacks in `cronUtils.js`). The initial implementation lacked private IP filtering during the actual socket connection and failed to block bypasses like `0.0.0.0/8` and IPv4-mapped IPv6 addresses (`::ffff:127.0.0.1`).
**Learning:** URL validation before a request is insufficient for SSRF protection because attackers can use DNS rebinding to change the resolved IP between the validation step and the actual HTTP connection. Also, Node.js `dns.lookup` with `autoSelectFamily` can return an array of addresses that must all be checked, and `0.0.0.0` or `::ffff:` can bypass standard loopback checks.
**Prevention:** Always implement a custom `http.Agent` and `https.Agent` that overrides `lookup` to validate the resolved IP address immediately prior to connection. Additionally, explicitly block the `0.0.0.0/8` network (`parts[0] === 0`) and IPv4-mapped IPv6 addresses (`::ffff:`) alongside standard private ranges.
