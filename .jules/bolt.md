## 2024-05-24 - Missing Debounce Causes Main Thread Blocking on Search Inputs
**Learning:** This vanilla JS frontend architecture (without a build step or external libraries) directly binds complex array filtering operations (`allGamesData.filter(...)`) to the `input` event on search fields. Because these operations are synchronous and run on every single keystroke, they block the main thread and cause noticeable lag, especially when typing quickly. The lack of a global module system means standard solutions like importing lodash aren't feasible without adding script tags.
**Action:** Always check for `addEventListener('input')` bindings that trigger data filtering or DOM updates in this vanilla architecture. Implement a local `debounce` utility function directly within the relevant scripts (`script.js`, `compare-prices.js`, `admin.js`, `validate.js`) to batch keystrokes and prevent synchronous blocking.
